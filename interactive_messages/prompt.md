# Interactive Messages Page Prompt

**目标**：重构 `interactive_messages/index.html` 的右侧控制面板，使其结构更清晰，直接对应两种主要场景。

**UI 结构变更**：
1.  **移除**：原有的单选框场景选择器和单一的消息按钮组。
2.  **新增**：将右侧控制区垂直分为两个独立的**场景区块 (Scenario Block)**。
    *   **区块 1：未打开 App 时 (System Push)**
        *   **主按钮/标题**：点击可切换左侧模拟器至锁屏/系统通知视图。
        *   **子按钮组**：包含 "评论"、"@我"、"点赞"、"新粉丝" 4个按钮。点击任意一个，不仅切换场景，还立即触发对应的系统通知演示。
    *   **区块 2：打开 App 未在当前页面 (In-App Notification)**
        *   **主按钮/标题**：点击可切换左侧模拟器至 App 内视图。
        *   **子按钮组**：包含 "评论"、"@我"、"点赞"、"新粉丝" 4个按钮。点击任意一个，不仅切换场景，还立即触发对应的应用内悬浮提示演示。

**新增需求 (2026-01-04)**：
1.  **文案调整**：
    *   逻辑说明标题：统一显示为 "互动类型：[类型]"，移除 "(系统通知)" 等后缀。
    *   逻辑说明内容：统一变更为 "点击可以跳转到相应的评论列表页，点击返回则返回到消息首页。最多展示20个字，超出...显示。图片显示[图片]。"
2.  **视觉调整**：
    *   **未打开 App 场景**：手机背景图更换为 `assets/desktop.jpg`。
    *   **通知标题**：系统通知的标题栏显示 `assets/tllogo.png` 图标 + "推栏" 文字。
3.  **交互功能**：
    *   **点击通知跳转**：点击系统通知或应用内气泡，需进入模拟器内的 "列表页"（而非单条详情页）。
    *   **消失动画**：当点击通知进行跳转时，通知元素本身需要执行一个消失动画（如向上滑动并淡出），动画结束后再显示列表页。
    *   **列表页**：
        *   标题：显示当前的互动类型（如 "评论"、"@我"、"点赞"、"新粉丝"）。
        *   内容：展示该类型的多条模拟消息列表（例如3-5条）。
        *   返回：点击返回按钮，返回到 "消息首页"（模拟为 App 内首页视图）。
4.  **内容处理**：
    *   **评论消息**：增加文案长度，并在展示时截断（最多20字，超出显示 `...`）。
    *   **图片处理**：如果内容包含图片，显示文本 `[图片]`。

**新增需求 (2026-01-04 补充)**：
1.  **设备模拟**：
    *   **屏幕尺寸**：适配 Xiaomi 15 Pro 尺寸（高瘦型，约 20:9 比例）。
2.  **未打开 App 场景 (System Push)**：
    *   **导航流程**：点击通知进入详情页 -> 点击返回 -> 进入 **消息首页**。
    *   **消息首页**：背景图使用 `assets/msgidx.jpg`。
3.  **打开 App 场景 (In-App)**：
    *   **界面精简**：移除头部 Header 和底部导航栏，仅显示背景图（`tlsq.jpg`）。
    *   **通知样式**：统一使用与 "未打开 App" 场景一致的 **顶部横幅通知** (Banner)，不再使用右下角气泡。
4.  **交互功能**：
    *   **点击通知跳转**：点击系统通知或应用内气泡，需进入模拟器内的 "列表页"（而非单条详情页）。
    *   **消失动画**：当点击通知进行跳转时，通知元素本身需要执行一个消失动画（如向上滑动并淡出），动画结束后再显示列表页。
    *   **列表页**：
        *   标题：显示当前的互动类型（如 "评论"、"@我"、"点赞"、"新粉丝"）。
        *   内容：展示该类型的多条模拟消息列表（例如3-5条）。
        *   返回：点击返回按钮，返回到 "消息首页"（模拟为 App 内首页视图）。
    *   **@我 消息卡片**：
        *   文案调整：在 "用户B 提到了你" 下方，新增具体的提及内容（如 "今天天气晴朗@慧慧"），形成两行展示效果。
    *   **打开 App 场景 (In-App)**：
        *   **卡片样式**：评论、@我、点赞、新粉丝的消息推送卡片内容中，均无需展示推栏标题和时间。

**交互逻辑**：
*   **评论消息内容格式**：
    *   **未打开App场景**：
        *   上方：显示推栏标题及时间。
        *   中间：显示 "用户A 评论了你"。
        *   最后一行：显示具体评论内容。
    *   **打开App未在当前页面场景**：
        *   上方：显示 "用户A 评论了你"。
        *   最后一行：显示具体评论内容。
        *   **注意**：无需展示推栏标题和时间。
    *   **截断规则**：具体内容超出20字用 `...` 显示。
*   **点赞消息内容格式**：
    *   在打开App未在当前页面场景中，内容显示为 "用户C 点赞了你"。
*   **新粉丝消息内容格式**：
    *   在打开App未在当前页面场景中，内容显示为 "用户D 关注了你"。
*   点击 **主按钮**：仅切换左侧模拟器的背景和状态，不触发具体消息。
*   点击 **子按钮**：
    1.  检查当前场景是否匹配，如果不匹配则自动切换场景。
    2.  在模拟器中展示对应的消息效果（系统通知条或应用内气泡）。
    3.  更新底部的逻辑说明文案，显示该特定场景下该类型消息的推送策略。

**代码结构要求**：
*   **JS 分离**：所有交互逻辑代码必须从 HTML 文件中剥离，存放在同级目录下的 `script.js` 文件中。
*   **引用**：在 `index.html` 底部通过 `<script src="script.js"></script>` 引入。
